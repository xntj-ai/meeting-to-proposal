一次客户访谈,
十几分钟变成完整提案

meeting-to-proposal 是一个 Claude Code 技能。给它一次客户访谈 —— 音频或会议记录 —— 它就端到端跑成一份可交付的提案网页:13 维度商业画布、5 维度个人 IP 定位、5 维度内容选题库。

它是一个 orchestrator:把录音切段转写、组织一支多智能体调研团队,再调用三个子技能分别撰写每一部分,最后合成一个隐去客户名称的 HTML 页面。沉淀自「张拼拼·XNTJ」EP0013 AI 帮我拿单子 的一次实战:一场 2.5 小时的新加坡客户访谈,约十分钟出一份完整提案。它的核心是思路而不是工具绑定。

三段流水线

从一段客户访谈,到一份隐去客户名的提案,中间是这条管线。

输入

客户访谈

访谈音频 +(可选)会议软件自带逐字稿。音频用更强模型重识别,逐字稿留下姓名与时间戳。

→
抽取

结构化

切段 ASR 转写,再由多智能体团队调研,把散乱的对话抽成可分析的字段。

→
撰写

调三个子技能

business-canvas、personal-ip-positioning、content-topic-design 各写一部分。

→
产出

提案网页

四区块合成一个 HTML,自动隐去客户名称,直接交付。

最佳输入是音频 + 逐字稿一起给 —— 逐字稿哪怕只有 60-70% 准确率,也保住了参会人姓名和时间戳;音频则让技能重新识别一遍。

提案的章节结构

三个子技能各自负责一块,合成页面里是四个核心区块。

商业画布 business-canvas

13 个维度全填:客户的客户、痛点、规模测算、商业模式、关键假设与验证。

个人 IP 定位 personal-ip

跨界三角:深度 × 跨度 × 特质,推出「只有他能讲」的差异化角度 + 5 个长期标签。

内容选题库 content-topic

5 维度 × 约 2-3 子方向 × 每子方向 10 选题 = 一份够跑半年的选题库。

商业运营分析 个人 IP 定位 自媒体规划 关键假设与验证路径

多智能体调研

转写文本进去,四个 agent 各管一段 —— 调研聚焦,不发散。

客户访谈转写文本
切段 ASR + 会议背景作上下文

结果交叉验证 不采信单一 agent 喂给三个子技能

关键认知:整段 ASR 在 5 分钟以上会注意力塌陷(EP0013 第一次跑 24 分钟,9 分钟就崩了)。切成 7-8 分钟一段、按偏移叠加时间码合并,识别率比会议软件自带逐字稿高 20-30 个百分点。

核心特性

端到端编排

一个技能把整条链路从原始录音一路带到成品提案网页,你不用自己拼接各步骤。

切段 ASR

7-8 分钟一段避开注意力塌陷,会议背景作上下文,识别率高过会议软件 20-30 个百分点。

三个可组合子技能

商业画布 / 个人 IP 定位 / 内容选题库,每个都能独立使用,字段表来自 EP0013 实战。

默认脱敏

合成页面写「一家做 XX 业务的公司」,只提业务类型,绝不点名客户。

适用场景

适合

有访谈,要整套提案

  • 咨询顾问 / 自由职业者 / 一人公司创始人 / BD / 售前
  • 手上有客户访谈音频或逐字稿,想出可交付的咨询报告
  • 要商业画布 + 个人 IP 定位 + 内容选题方案一整套
不适合

纯整理 / 无咨询输出

  • 只想把逐字稿整理干净 —— 用 ASR 工具即可
  • 不产出咨询内容的客户回访
  • 纯 RFP 应答

快速开始

装进 Claude Code 技能目录

git clone https://github.com/xntj-ai/meeting-to-proposal.git ~/.claude/skills/meeting-to-proposal

把客户访谈扔进对话

把音频(最好再配上会议软件逐字稿)扔进对话,说"用 meeting-to-proposal 把这次访谈做成完整提案",Claude 会自动触发并跑完 7 步编排。

拿到隐去客户名的提案网页

四区块合成的 HTML 直接交付。也可不经 Claude,把 SKILL.md 粘进任意 LLM,核心是思路不是工具。
