One client interview,
a full proposal in minutes

meeting-to-proposal is a Claude Code skill. Hand it a client interview — audio or a meeting transcript — and it runs end to end into a deliverable proposal web page: a 13-dimension business canvas, a 5-dimension personal IP positioning, and a 5-dimension content topic library.

It is an orchestrator: it chunks and transcribes the recording, runs a multi-agent research team, calls three sub-skills to write each part, then synthesizes one HTML page with the client name redacted. Distilled from a real run on 「张拼拼·XNTJ」 EP0013: a 2.5-hour interview with a Singapore client turned into a full proposal in about ten minutes. The method is the thinking, not a tool lock-in.

The best input is audio + transcript together — the transcript keeps attendee names and timestamps even at 60–70% accuracy, while the audio lets the skill re-recognize the speech.

The proposal's section architecture

Three sub-skills, each owning one block; the synthesized page has four core sections.

business-canvas 商业画布

All 13 dimensions filled: customer's customer, pain points, market sizing, business model, key assumptions & validation.

personal-ip IP 定位

The cross-discipline triangle: depth × breadth × traits → the angle only they can take, plus 5 long-term tags.

content-topic 选题库

5 dimensions × ~2–3 sub-directions × 10 topics each = a topic library good for six months.

The key insight: single-pass ASR over 5+ minutes triggers attention collapse (a 24-minute pass broke around the 9-minute mark in the EP0013 run). Splitting into 7–8 minute segments and merging with offset timestamps runs 20–30 points above the meeting software's built-in transcript.

Core features

End-to-end orchestration

One skill carries the run from raw recording all the way to a finished proposal page — you do not stitch the steps together.

Chunked ASR

7–8 minute segments avoid attention collapse, meeting background as context, 20–30 points above the meeting software.

Three composable sub-skills

Business canvas / personal IP / content topics, each usable on its own, with field tables from the EP0013 run.

Redaction by default

The synthesized page writes "a company in the XX business" — business type only, never the client name.

When to use

Good fit

Have an interview, want the full package

  • Consultants, freelancers, one-person-company founders, BD, pre-sales
  • An interview recording or transcript you want turned into a deliverable report
  • The whole set: business canvas + personal IP + content plan
Not for

Cleanup only / no consulting output

  • Just cleaning up a transcript — use an ASR tool
  • A client check-in with no consulting output
  • A pure RFP response

Quick start

Install into your Claude Code skills directory

git clone https://github.com/xntj-ai/meeting-to-proposal.git ~/.claude/skills/meeting-to-proposal

Drop a client interview into the chat

Drop the audio (ideally with the meeting software's transcript) into the conversation and say "use meeting-to-proposal to turn this interview into a full proposal" — Claude triggers it and runs the 7-step orchestration.

Get a proposal page with the client redacted

The four-section HTML is ready to hand off. Or skip Claude: paste SKILL.md into any LLM — the method is the thinking, not the tool.
