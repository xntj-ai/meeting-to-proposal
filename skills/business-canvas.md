# business-canvas · 商业画布分析

> 基于「张拼拼·XNTJ」EP0013 公开 demo 的实际产出反推。
> 13 个分析维度，覆盖客户访谈到完整商业画布的端到端字段。

## 何时调用

被 [`meeting-to-proposal`](../SKILL.md) orchestrator 在 Step 4 自动调用；也可以独立使用 —— 用户给出客户/公司的业务描述 + 行业背景，要求做商业画布分析时。

## 输入

| 字段 | 必要 | 说明 |
|---|---|---|
| 公司业务描述 | ✓ | 一段话讲清楚他在做什么 |
| 创始人/受访者背景 | ✓ | 来自客户访谈（成长经历 / 职业生涯）|
| 行业背景 | ✓ | 由多智能体团队调研提供 |
| 公司发展阶段 | 可选 | 早期 / 成长期 / 成熟期 |
| 客户目标市场 | 可选 | 国内 / 出海 / 海外回流 |

## 输出 · 13 个分析维度

每个维度都是商业画布的一个 box，按顺序输出：

### 1. 客户的客户

> 不是「我们的客户是谁」，是「我们客户的客户是谁」 —— 这是真正决定生意成立的人。

- 谁掏钱（客户）
- 谁享受价值（客户的客户）
- 谁负责决策（决策链）

### 2. 痛点

- 客户当下最痛的 1–3 件事
- 痛的严重程度（影响营收 / 影响效率 / 影响合规）
- 现有解决方案 vs 我们解决方案的差距

### 3. 价值收益

- 量化的价值（节省时间 X 小时 / 增收 Y 万 / 减少风险 Z%）
- 非量化的价值（确定性 / 安全感 / 专业信任）
- 客户愿意为此付多少（参考客单价）

### 4. 需求人群细分（三个垂直客群）

强制拆出 **3 个垂直客群**（不多不少），每个客群独立分析：

| 客群 | 画像特征 | 触达渠道 | 客单价范围 |
|---|---|---|---|
| 客群 A | … | … | … |
| 客群 B | … | … | … |
| 客群 C | … | … | … |

### 5. 市场研究

- **宏观趋势**：行业未来 3–5 年的方向
- **竞争格局**：头部 / 腰部 / 长尾的分层

### 6. 竞争优势

- 直接竞品（同质化对手）
- 间接竞品（替代方案）
- 我方差异化优势（基于受访者的专业背景 + 行业经验）

### 7. 规模测算

三层市场漏斗：

- **TAM 潜在市场**（Total Addressable Market）
- **SAM 可服务市场**（Serviceable Available Market）
- **SOM 可获得市场**（Serviceable Obtainable Market，3 年内）

### 8. 核心卖点 / 解决方案 / 产品内核

- 一句话核心卖点（用客户的语言）
- 解决方案的具体形态
- 产品内核（背后真正的竞争壁垒）
- 交付流程（客户体验路径）

### 9. 商业模式 · 单元模型

- 收费方式（一次性 / 订阅 / 抽成 / 混合）
- 单元经济（每单成本 / 每单收益 / 毛利率）
- 核心指标（最关键的 3 个）

### 10. 获客增长

针对国内主战场，给出 2-3 个平台的转化链路：

- **视频号转化链路**：从内容 → 加企业微信 → 私域 → 转化的完整步骤
- **小红书转化链路**：从搜索 / 推荐 → 主页 → 私信 / 评论 → 转化的步骤
- 其他平台（视情况）

### 11. 构建壁垒 / 合作伙伴和渠道

- 数据壁垒 / 渠道壁垒 / 品牌壁垒 / 网络效应（任选 1-2 个适用的）
- 关键合作伙伴（供应商 / 渠道商 / 联盟伙伴）

### 12. 关键假设

把生意成立的「必须为真」的假设全部列出来：

- 市场假设（市场足够大 / 增速足够快）
- 客户假设（客户愿意为此付费 / 客户能找到我们）
- 产品假设（我们能交付承诺的价值）
- 竞争假设（我们的差异化足以抵御竞争）

### 13. 如何测试和验证

每个关键假设给出**最小成本测试方法**：

- 假设 X → 用 1 个 MVP 在 2 周内验证
- 假设 Y → 用 5 次客户访谈在 1 周内验证

## 输出格式

最终交付一份 HTML 网页（或 Markdown），按上述 13 个维度组织，每个维度独立 section。前 12 是分析，第 13 是 action plan。

**约束**：

- 隐去真实客户名称（写「一家做 XX 业务的公司」即可）
- 数据测算要标注来源（即使是估算，也要写明估算逻辑）
- 不引用未经验证的行业报告数据

## 调用提示词

```
请你利用 business-canvas skill，基于以上调研结果，对这家公司做完整的商业画布分析。
按 13 个维度输出，每个维度独立 section。客户名称隐去。
```
