# /meeting-to-proposal

把一次客户会议，十几分钟变成完整咨询提案。

> 沉淀自张拼拼·XNTJ [EP0013 AI 帮我拿单子](https://xntj.tv/ep/live-ep0013/)。把过往每次咨询会议的 input + output 作为最佳实践参考，用最好的模型把专业能力萃取出来。

## 核心理念

客户开会、梳理需求、提案 —— 不管你是百万年薪高管、OPC、还是创业者，这是日常业务最头疼的环节。这个 skill 把它压缩到十几分钟：

```
客户访谈音频 → 转写 → 多智能体调研 → 调用 3 个子 skill → 提案网页
```

**关键认知**：客户只是商机，不是确定性客户。用 AI 把过往经验高效给到他 → 快速筛选 → 信任 → 长期合作。这个 skill 的本质是把你的专业能力**蒸馏**成可重复调用的形态。

## 7 步 orchestration

### Step 1 · 准备会议素材

**输入**：以下任一：

- 客户访谈音频文件（在线会议软件导出，常见 `.mp3` / `.m4a` / `.wav`）
- 已有的会议记录文本（在线会议软件自带的逐字稿，正确率通常 60-70%）

**最佳输入**：**纯音频 + 已有的会议记录文本** 同时给。已有文稿留下了参会人姓名 + 时间戳，纯音频可以让我们用更先进的模型重新识别。

### Step 2 · 切段 ASR 转写

**注意 5 分钟+ 整段 ASR 会触发注意力塌陷**（EP0013 第一次跑 24 分钟，9 分钟时识别崩坏）。

```
用 FFmpeg 切音频成 7-8 分钟一段
每段独立 ASR
合并时按段偏移叠加时间码
```

**重要**：开始转写前，把**会议背景信息** + **参会人** + **讨论方向**作为上下文喂给 ASR 模型。这是它能识别准的关键 —— 比普通在线会议软件好 20-30 个百分点。

详细方法参考 [`live-video-postprod`](https://github.com/xntj-ai/live-video-postprod) skill 的 § 01 章节，切段 ASR 同套方法论。

### Step 3 · 多智能体团队调研

把转写文本扔给 Claude Code，提示词模板：

```
现在给你看到的这个文本文件是我跟一个客户的现场沟通，我准备了很多问题对他进行了采访。包括他的成长经历和个人的职业生涯，以及他现在公司的客户和业务。

我这么做的目的是为了帮他去完整地分析他在国内的商业模式，以及整个的商业画布。请你组织一个多智能体的团队，对这个生意的模式进行全方位的调研。

那从调研的结果当中，利用我关于商业画布分析的这个经验，去完整地详细地分析它在国内的商业模式以及它的这个客户的情况。

那同时我们也要去梳理他的个人的成长经历，包括他的一些这个个人的特点特质。

那接下来我会需要对它做自媒体这件事情做一个完整的定位，以及帮他去分析他可能的话题维度，以及去做一些选题的示例。

那这个方案我在我的 skill 里面也有对应的经验可以调用。

如果你完成了这样的一个信息的梳理之后呢，我需要你最终以一个网页的形式对我进行一个完整的呈现。

那因为这是一个真实的案例，那我需要你在网页当中去隐去客户的名称，只提及它是一个什么样的业务就好了。
```

**多智能体团队建议拆分**：

| Agent | 职责 |
|---|---|
| 行业调研员 | 宏观趋势 / 市场规模 / 竞争格局 |
| 竞品分析员 | 直接竞品 / 间接竞品 / 差异化机会 |
| 客户拆解员 | 客户的客户 / 痛点 / 三个垂直客群 |
| 数据测算员 | 潜在市场 / 可服务市场 / 可获得市场 |

### Step 4 · 调用 business-canvas 子 skill

详见 [`skills/business-canvas.md`](./skills/business-canvas.md)。

输出：完整商业画布分析报告，13 个分析维度，全部按 EP0013 实际 demo 的字段表。

### Step 5 · 调用 personal-ip-positioning 子 skill

详见 [`skills/personal-ip-positioning.md`](./skills/personal-ip-positioning.md)。

输出：基于客户访谈中的成长经历 / 职业生涯 / 个人特质，做出差异化定位。

### Step 6 · 调用 content-topic-design 子 skill

详见 [`skills/content-topic-design.md`](./skills/content-topic-design.md)。

输出：5 个内容维度 × 每维度子方向 × 每子方向 10 个选题。

### Step 7 · 合成最终提案网页

把三个子 skill 的输出合成一个 HTML 提案网页。约束：

- **隐去客户名称**（真实案例，写「一家做 XX 业务的公司」即可）
- 用 PPVI 浅色版美学（参考 [methodology.html](https://github.com/xntj-ai/live-video-postprod/blob/main/methodology.html) 的视觉规范）
- 4 个核心区块：商业运营分析 / 个人 IP 定位 / 自媒体规划 / 关键假设与验证路径

## 为什么十几分钟能跑完整方案

不是模型变快了，是**你的专业能力被蒸馏过**：

- **input**：过去每一次咨询会议的内容（包括会前准备问题，甚至会前问题都是 AI 准备的）
- **output**：每次会后给客户的报告 — 作为**最佳实践参考**
- **process**：怎么搜索调研、怎么思考、怎么一步步输出报告 — 写进了 skill

给完这些素材之后，**把对商业画布分析、个人 IP 定位、选题确定的专业技能蒸馏成 skill** — 完成了这个过程。

## 关键约束

1. **AI 是赋能不是取代** — 它能做到今天的效果，建立在你的专业性的基础上
2. **客户只是商机** — 这个流程是为了 **快速筛选 + 建立信任 + 长期合作**，不是替代真实的服务交付
3. **真实案例的脱敏要求** — 客户名称必须隐去，只提业务类型
4. **多智能体调研结果要交叉验证** — 不能直接采信单一 agent 的输出
